# 6_full_rebrand_russian_marketplace

## Что было сделано

- Выполнен ребрендинг фронта: обновлены визуальные токены, поверхности, акцентные цвета и состояния.
- Ключевые пользовательские разделы переведены на русский язык.
- Добавлены UX-анимации:
  - переключение login/register;
  - health-пульсация статусов;
  - плавные переходы в auth-форме.
- Реализован каталог GPU Pods и шаблонов:
  - backend таблицы `pods_catalog`, `pod_templates`, `pod_template_bindings`;
  - seed 30+ pods и 5 шаблонов;
  - публичные catalog endpoints и admin CRUD.
- Реализованы планы аренды серверов:
  - `rental_plans`, `server_orders`;
  - расчет стоимости и создание заказа;
  - frontend-конфигуратор hourly/monthly с выбором ОС/сети/ресурсов.
- Реализованы персонализация и SSH ключи:
  - `user_settings`, `user_ssh_keys`;
  - CRUD endpoints в authservice;
  - UI раздел «Настройки».
- Реализован прямой вход `/admin`:
  - endpoint `POST /v1/auth/admin/direct` с `admin/admin123`;
  - аудит попыток входа в `admin_access_logs`.
- В админку добавлен one-command блок установки агента:
  - endpoint `GET /v1/admin/agent/install-command`;
  - вывод `curl` команды в админ-консоли.
- Обновлен release flow:
  - сборка hostagent бинарников (linux/darwin/windows);
  - загрузка бинарников в GitHub Release assets через `Makefile`.

## Почему так сделано

- Проекту требовался переход от базового control-plane UI к продуктовой модели marketplace.
- Без backend-сущностей каталог/аренда/SSH невозможно реализовать без моков.
- `/admin` доступ и аудит были выделены в отдельный auth-контур для явного контроля и трассировки.

## С какими трудностями столкнулся

- Нужно было сохранить совместимость с существующими API и уже внедренным RBAC.
- Большой объем UI-текста требовал поэтапной русификации без регрессий.
- Расширение `Makefile` под multi-platform binaries требовало верификации артефактов в release-пайплайне.

## Какие решения принял

- Каталог Pods сделан в hybrid-модели: seed + редактирование через админ API.
- Расчет аренды серверов реализован server-side в billingservice.
- Команда one-command установки формируется сервером админки на основе `GITHUB_REPO` и `RELEASE_TAG`.

- Добавить e2e сценарии под новые контуры: `/admin` login, pods catalog, server rental, SSH settings.
