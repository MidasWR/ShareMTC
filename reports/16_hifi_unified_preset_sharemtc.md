# Итерационный отчёт: Hi-fi unified preset для ShareMTC/MTS Space

## Что было сделано

- Собран единый dark-first UI preset для ключевых потоков: `Auth`, `Marketplace/Deploy`, `My Instances`, `Instance Details`, `Settings`.
- Обновлены дизайн-токены в группах `Canvas/Surface/Elevated/Border/Text/Brand` с MTS red как главным акцентом.
- Удалён альтернативный neon-бренд, заменён на `mono`-вариант без зелёных/синих акцентов в логотипе.
- Внедрён icon primitive и унифицированы размеры stroke-иконок (`16/20/24`) в новых сценариях.
- Реализован hi-fi `Marketplace / Deploy`:
  - верхняя панель фильтров (compute type, country multi-select, VRAM slider, price range, availability, network),
  - каталог tile-карточек,
  - правый `Deploy wizard` на 7 шагов с `Review & Deploy`.
- Реализован hi-fi `My Instances`:
  - data-dense список с быстрыми действиями,
  - expandable строки с endpoints/SSH/events/billing summary.
- Реализован `Instance Details` с вкладками `Overview/Logs/Metrics/Billing` и empty/error состояниями.
- Обновлён `Settings`:
  - RU/EN copy для ключевых зон,
  - language/theme/brand/timezone,
  - SSH keys CRUD с подсказками.
- Добавлен набор мок-данных из 12 инстансов с требуемыми странами, размерами, статусами, ошибками и экономикой.

## Почему так сделано

- Основной бизнес-флоу compute-marketplace — это быстрый выбор, понятная конфигурация и предсказуемое управление.
- Side-panel wizard снижает ошибки при deploy и даёт прозрачный контроль стоимости/рисков на этапе review.
- Expandable-операционный список ускоряет ежедневные действия без постоянных переходов между экранами.
- Единые токены и унифицированная иконография снижают визуальный шум и повышают консистентность всей консоли.
- RU/EN и форматирование валюты/дат встроены в ключевые сценарии, чтобы избежать смешения локалей.

## С какими трудностями столкнулся

- Историческая структура проекта содержала mix старых и новых UX-паттернов, требовалось сохранить совместимость.
- Роутинг и текущие панели уже были связаны с демо-логикой, поэтому некоторые сценарии пришлось модернизировать inplace.
- Нужно было убрать neon-наследие аккуратно, не ломая текущую модель пользовательских настроек.

## Какие решения принял

- Перевёл бренд-тему на `mts + mono`, сохранив переключение через settings provider.
- Для покрытия запроса по deliverables без моков-заглушек добавил реалистичный data module с 12+ инстансами.
- `Instance Details` встроен как полноценный блок с вкладками в рамках `My Instances`, чтобы flow оставался непрерывным.
- Состояния empty/error вынесены в единый паттерн с явным `kind`.

## Планы на следующую итерацию

- Довести покрытие RU/EN до 100% по всем панелям через централизованные словари.
- Добавить визуальные regression checks для desktop/tablet/mobile breakpoint-срезов.
- Подключить e2e-сценарий на deploy wizard + lifecycle actions + SSH keys CRUD.
- Синхронизировать mock/demo статусы с backend контрактом и telemetry-метриками в реальном времени.
